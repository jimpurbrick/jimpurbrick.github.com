The Creation Engine No. 2

crestmatic

 Sun 03 January 2016 by Jim Purbrick

A year ago I gave a talk at EVE Vegas about building RESTful CREST applications. My #1 recommendation was to specify representations in requests, but that’s hard to do when there is little documentation on which representations are available and what they contain.

Fortunately CREST is that it’s self describing: you just need to send an OPTIONS request to a CREST URI and the API will send a list of methods and representations that can be used with that URI. The data returned isn’t in a standard format, so I built the crestschema library which can convert the CREST format in to a more useful JSON schema format. The library is designed to work both in browsers, in applications like crestexplorer, and from node. From node applications like crestschemaspider can crawl the CREST API to find representations which can be converted to JSON schema and saved to disk. The schemas can then be used with wide variety of software, libraries and languages to validate data from the live CREST API, generate parsers or automatically generate documentation.

With the JSON schemas collected it was possible to build crestmatic which uses crestschema and matic to generate documentation. The final steps were to add a travis step to publish the generated documentation to the crestmatic gh-pages branch and to use nightli.es to trigger a build every night to ensure that the documentation is kept up to date. Please feel free to fork the repo, submit a pull request if you’d like to see some changes or just donate some ISK to Capt Out if you find the documentation or code useful.
